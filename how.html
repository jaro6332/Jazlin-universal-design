How can you impliment universal design?

WCAG

The Web Content Accessibility Guidelines (WCAG) 2.0, developed by the World Wide Web Consortium (W3C), are a set of internationally recognized standards for making web content more accessible to people with disabilities. These guidelines are critical for ensuring that websites, applications, and digital documents are usable by everyone, including individuals with visual, auditory, cognitive, or physical disabilities.

The Four Principles of WCAG 2.0

POUR

  • Perceivable - Users must be able to perceive the information being presented (it can't be invisible to their senses). Examples include,
      • Provide text alternatives for non-text content (e.g., alt text for images).
      Offer captions for videos and transcripts for audio content.
      Use sufficient contrast between text and background colors for readability.
  • Operable - Users must be able to operate the interface, regardless of the device or assistive technology they use. Examples include,
      • Ensure all functionalities are accessible via keyboard navigation.
      Avoid designing elements that could trigger seizures, such as flashing content.
      Provide clear navigation mechanisms, such as consistent menus.
  • Understandable - Content must be presented in a clear and predictable way, so users can comprehend it. Examples include,
      • Use plain language where possible and define technical terms.
      Ensure forms have clear labels and instructions.
      Provide feedback for user errors and suggestions for correction.
  • Robust - Content must be robust enough to work with a wide variety of technologies, including assistive devices. Examples include,
      • Use clean and valid HTML and CSS for compatibility.
      Test your site with screen readers and other assistive technologies.
      Ensure dynamic content updates are communicated to assistive technologies.

Each criterion also falls into one of three levels of compliance: A, AA, and AAA representing minimum, must have, and good to have respectively. For a list of all success criteria, including documentation on understanding and implementing them, visit the WCAG official guidelines.

Coding practices

The best way to ensure a usable site is by implementing accessible coding practices.

Semantic structure

Accessible code begins with making sure every item on the site serving any informational or interactive purpose (a.k.a. any purpose other than design) is defined semantically. This is necessary for browsers, search engines, and assistive technology to properly interpret your website.

To properly implement semantic structure:

  • Regions (header, footer, nav, main, etc.) should be properly defined
  • Headings should be used in ascending order to properly represent their hierarchical meaning
  • Use the correct list type for the information you are trying to convey
  • Use <section> to group related content and <fieldset> to define a group of form elements (a.k.a widgets) with a shared purpose
  • Use labels to properly define input fields
  • Use headers in tables to define row and columns
  • Utilize ARIA (Accessible Rich Internet Applications) to provide semantics or define interactions with assistive technology that HTML cannot define. Remember the first rule of ARIA is to use HTML instead unless it's impossible to provide accessibility without it

Separation of content and design

Separating the design of a page from its content allows users to configure your site to meet their needs without impacting the information. To do this, all styling should be done only in CSS while the HTML holds only the content. A great example of this is CSS Zen Garden which displays the same HTML content many different ways by using different styling files. Separating your styling for your content will also benefit your future maintenance as the website can be entirely redesigned without affecting the content and vice versa.

Responsive design

Responsive Design is web design that changes in response to a device's screen size. Responsive design can be implemented by using a grid-based CSS layout, flexible media, and CSS media queries to define behavior for certain screen sizes. It is recommended to build mobile first and practice progressive enhancement, using the main style sheet to define mobile design and media queries to define behavior for larger screen sizes, but desktop-first graceful degradation will also suffice.

Accessibility Testing tools

There are various tools you can use to test the accessibility of your site:

  • Wave & Silktide - These browser extensions help identify accessibility issues on your site. WAVE highlights errors directly on the page, while Silktide offers additional features, such as an in-browser screen reader, simulations for vision impairments and dyslexia, and a focus order overlay to analyze navigation.
  • Validator.nu - Based on the W3C validator, this tool allows you to assess HTML for semantic accuracy and accessibility. It supports both live URLs and local code files.
  • Adobe Color - This online tool assists in selecting color palettes that meet WCAG AA and AAA contrast standards. It also checks if your color choices are safe for users with color blindness.
  • Outline extension - This extension generates a semantic outline of your website, helping you verify that the structure is logically organized for assistive technologies.
